# CUDA Freshman 课程持久化记录

本目录用于长期记录学习进度、互动内容、实验结果和 Agent 的教学决策。设计原则是：状态文件机器可读，日志文件人类可读，章节笔记便于复盘，实验结果可追溯。

## 文件结构

```text
course_records/
  README.md
  learner_profile.yaml
  progress.yaml
  session_log.md
  chapters/
    00_environment.md
    01_hello_world.md
```

## 记录分层

`learner_profile.yaml` 记录相对稳定的信息，例如学习目标、GPU 环境、偏好的学习节奏和已知基础。Agent 只有在用户明确提供新信息或环境发生变化时才修改它。

`progress.yaml` 记录当前课程状态，例如当前章节、每章状态、最近一次命令、阻塞点、下一步建议。Agent 每次正式带课前应先读取它，每次完成一个互动小循环后应更新它。

`session_log.md` 是按时间追加的互动日志，记录每次学习会话的目标、命令、观察、解释、结论和后续动作。它不追求记录逐字对话，而是记录可复盘的学习证据。

`chapters/*.md` 是章节级复盘笔记，适合沉淀每章核心概念、用户误区、实验结果、待回看问题。章节完成时必须更新。

## Agent 更新规则

1. 每次开始授课先读取 `progress.yaml`，确认当前章节、阻塞点和下一步建议。
2. 如果用户提供硬件、目标、时间安排等长期信息，更新 `learner_profile.yaml`。
3. 每完成一次“预测 -> 执行 -> 观察 -> 解释 -> 改动 -> 复盘”循环，更新 `progress.yaml` 并向 `session_log.md` 追加一条记录。
4. 每章结束时更新对应章节笔记，把“会运行”提升为“会解释”的证据写进去。
5. 不把大段命令输出完整粘贴进日志，只记录关键行、结论和必要的错误信息。
6. 如果实验失败，保留失败记录，标注下一步排查方向，不要覆盖成成功结果。
7. 修改记录时保持 YAML 字段稳定；新增字段可以追加，已有字段含义不要随意改变。

## 状态枚举

章节状态建议只使用以下值：

- `not_started`：尚未开始。
- `in_progress`：正在学习。
- `blocked`：因环境、编译、运行或概念问题暂停。
- `review_needed`：已跑通但解释不充分，需要复习。
- `completed`：能独立运行、指出关键代码、解释核心现象。

掌握程度建议使用 0-4：

- `0`：未接触。
- `1`：能跟着运行。
- `2`：能解释主要 API 和数据流。
- `3`：能解释性能现象并做小改动。
- `4`：能迁移到新问题或独立调试。
